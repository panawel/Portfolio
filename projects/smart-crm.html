Project Scope

The Smart CRM company and its development team provided an APK file of the upcoming version (pre-release) for testing to achieve established business goals:

  • Focus on new features not available in the current Play Store version.
  • Ensure reduction in defects and bugs that could lead to failures and disruptions in user experience.
  • Avoid expensive post-release defect fixing.

The Testing Process - B2B Application

The execution phase was divided into 2 rounds:

Round 1

Functional Testing

Localization, GUI, Compatibility, End-to-End, Usability, Exploratory, CRUD.

Round 2

Non-Functional Testing

Integration, Authorization, Network, I18N, Interruption, Installation Testing, Backup & Restore, Security.

E2E (End-to-End)

The following business processes were tested:

  • Adding a new customer as a "lead" (creating a new lead for a potential customer).
  • Editing an existing lead details.
  • Sending a WhatsApp message to the customer without adding them to the contact list.
  • Purchasing a subscription and checking feature limitations based on subscription type.
  • Negative testing to challenge the system's behavior in boundary cases.

Integration

We designed and executed scripts to test integration between Smart CRM and third-party apps, ensuring correct data flow:

  • Integration between "Smart CRM" and the device browser.
  • Integration with WhatsApp to send pre-written message templates.

Gateway

  • Verified calculation of payment amounts based on premium plans, discounts, or promotions.
  • Ensured the subscription purchase process aligns with the Play Store purchasing process without bugs.

I18N (Internationalization)

We spoofed locations and changed device languages to test global compatibility:

  • Verified "Leads" filtering screen adaptation.
  • Checked subscription purchase screen adaptation and process success.
  • Ensured "Floating Circle" widget pop-ups display correctly.
  • Validated application menus, contact screens, and more.

Backup & Restore

Ensured validity, reliability, and integrity of the new backup/restore feature involving cloud data transmission.

We verified that the backup process preserves all required data and that restoration works after a server crash or device switch. We tested across different user accounts, subscription types, and devices to mitigate data loss risks.

C.R.U.D

Tests on database operations (Create, Read, Update, Delete):

  • User registration and account creation.
  • User login authentication and account access.
  • Subscription purchase process and account detail updates reflected across the app.

Interruption

Challenged the application with external disruptions to prevent crashes or data loss:

  • Triggering voice dictation while playing music.
  • Triggering voice notes during a phone call with Bluetooth headphones.
  • Testing collisions between the "floating circle" widget and other app widgets.
  • Responding to notifications from the notification shade.

Security

Conducted system resilience tests against unauthorized access and checked for privacy issues or data leaks.

We monitored and analyzed communication between the app and servers, verified encryption of backed-up data, and ensured no sensitive information was exposed during transfer.
