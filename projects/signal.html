Signal

Increasing the number of tests by 50% within a given timeframe

In my first year as a QA engineer, my curiosity about the world of QA grew, along with my desire to learn and explore the field of testing. I read about various types of testing and techniques, and I self-learned an AI tool called 'Maestro' and combined it with 'Appium' to utilize it for mobile automation testing of 'Signal' app. This allowed me to increase the number of tests that could be completed within the same timeframe, alongside manual testing.

Python Appium Maestro Mobile Automation Privacy

Overview

'Signal' is a messaging app focused on privacy. It's free, easy to use, open-source, and features strong end-to-end encryption to keep communications completely private.

Project Scope

The development team provided me with an APK file of our internal 'Signal' version, which I tested before it was rolled out to our customers worldwide:

Integration Features

  • Focus: New integration between Signal and internal classified products.
  • Tested unique features unavailable in the Play Store version.
  • Ensured no functionality loss during integration.

End-to-End Tests

  • Scenarios: User registration, OTP verification, messaging.
  • Features: Audio/Video calls, groups, media sharing.
  • Verified app scalability in split-screen/landscape modes.

Component Testing

  • Breakdown: Screen-by-screen element analysis.
  • Documentation: Detailed JIRA reports with reproduction steps & media.
  • Verified functional behavior of individual UI components.

UI Testing

  • Visuals: Buttons, text fields, layout correctness.
  • i18n: Validated display in different languages.
  • Ensured responsiveness across diverse devices.

Installation Testing

  • Lifecycle: Install, Update, Uninstall.
  • Verified stability across different OS versions and device types.

Regression & Exploratory

  • Regression: 600+ test cases run per new version/fix.
  • Exploratory: Creative usage to uncover edge-case defects.
  • Ensured new features didn't break existing core functions.

Automation

The efficiency and coverage of my testing efforts was greatly improved by basic automation tests. I automated a lot of test cases for Signal application using 'Maestro' AI tool which was also used together with 'Appium'. This included automating repetitive test cases such as message sending and receiving, calls, groups, user registration processes, and more...

I wrote python scripts to simulate various end-user scenarios to ensure that updates didn't affect the app's functionality.

This helped shorten regression testing periods and allowing new builds to be quickly checked.

Automated Android testing with direct Jira failure reporting.
