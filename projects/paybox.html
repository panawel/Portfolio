Paybox

Comprehensive QA support for Paybox - a leading mobile payment app with 3.2 billion ILS transferred in 2021.

FinTech Mobile App QA Engineering API Testing

Overview

I Developed and executed test plans for the "Voucher Carousel" Component. Using the 'Postman' tool, I executed API Tests, preventing critical bugs that could cause financial losses for over 1.5 million users.

The voucher carousel is the part of the app that allows users to:

  • Earn points with every purchase and convert them into money.
  • Send various types of vouchers as holiday or birthday gifts from the user's account to friends who also have the app installed.
  • Pay with the accumulated vouchers at selected supermarkets, stores, and Brands.

Voucher Carousel Feature

API Testing

In the "real world", actions such as redeeming a voucher, refund, or canceling sending to a friend, are typically API calls triggered by the system at the checkout counter of a store.

However, in the testing environment, I used the SDK version of the app and utilized Postman to simulate these actions.

By sending API requests through Postman, I could replicate these scenarios and verify that the system behaves correctly. I used Postman to test the voucher redemption process, ensuring that when a voucher is redeemed, it correctly updates the system and processes the payment. Similarly, I tested the refund and canceling sending to a friend process to ensure that these actions were handled smoothly and the system reverted the changes accurately.

UI Testing

I conducted UI tests covering all scenarios and screens where the vouchers are displayed. This was done to ensure the correct display of all 22 types of vouchers and to verify that the information is presented without distortion, and that no elements overlap or hide each other.

C.R.U.D & Database Integration

The application is in its maintenance phase, so occasionally, calls are received from customers who have made transactions and purchases using the app, experiencing various issues or errors. As a QA engineer with access to the customer database, I am responsible for tracing the steps and actions performed by the customers based on the information recorded in the company's database. This involves integrating between several databases to try to identify and understand the bugs that ultimately led to the failures. This is done to ensure that if an issue did occur, it is confirmed and promptly fixed in collaboration with the developers.

Integration Testing

I ensured that every user operation, such as purchases and transactions, was accurately and consistently reflected in the database. This included verifying that data was correctly written, updated, and retrieved from the database without discrepancies.

End-to-End Tests

I performed End-to-End tests focusing on Boundary Value Analysis and ensured vouchers were split when their amounts exceeded 1500 ILS.

For example, a voucher of 3000 ILS should split into two 1500 ILS vouchers, and a 3100 ILS voucher should split into two 1500 ILS vouchers and one 100 ILS voucher. This involved testing edge cases around the 1500 ILS boundary to confirm that the splitting logic functioned accurately and consistently.

Additionally, I tested vouchers with an identification mechanism requiring activation before use. I verified that if a user entered the wrong identification code three times, the voucher would be blocked. This included testing the correct activation process and the blocking mechanism to ensure the system effectively prevented unauthorized use after repeated incorrect attempts.

Exploratory Testing

I performed exploratory tests by interacting with the app in creative and unconventional ways to identify potential issues that might not be detected by formal testing methods. This involves thinking outside the box and experimenting with different features, navigation paths, and user interactions to uncover hidden bugs and usability problems.
