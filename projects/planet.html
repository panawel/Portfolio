Provided QA support for 'PLANET' premier cinema chain of the ticketing services.

As part of the QA course we were asked to perform professional tests, ensuring 'bug-free operation' of online ticket purchasing application.

Ticketing Services Web Application Accessibility

Overview

'PLANET' offering a world-class experience with the finest cinema complexes and latest innovations that provide an exceptional viewing experience, from giant screens, advanced sound systems and luxurious roomy seating to the most advanced ticketing services in the world and maximum accessibility for disabled persons.

Services Offered

Functional Tests
Compatibility Tests
Load Tests
I18N Tests

Project Scope

To cope with the project, a team of 3 QA engineers and 1 manager were assigned, the manager was responsible for quality level within software product:

A web application operating on HTML 5, cross-platform, devices and Browsers. It's functionality is 'seating management' system, so that the client can choose which seats in the theater they want to sit on in advance, and to pay for the tickets and seating positions using the application.

After studying the requirements, writing test cases, the engineers provided the following QA support:

Functional Testing

  • Scope: GUI, E2E, Accessibility.
  • Coverage: Verified annual ticket purchasing flow end-to-end (positive & negative).
  • Finding: Critical defect found in Venue 13 (seating plan mismatch vs reality), potentially causing revenue loss.

Accessibility

  • Tools: Accessibility Toolbar.
  • Tested all toolbar options ensuring compliance with Israeli Internet Association standards.

Compatibility Testing

  • Goal: Consistent UX across browsers/OS.
  • Executed E2E flows on various devices/browsers to detect cross-platform defects.

Load Testing

  • Tool: JMeter.
  • Simulated concurrent users to find capacity peaks.
  • Benchmarked performance against competitor cinema websites.

I18N Testing

  • Languages: Hebrew & English.
  • Defect: Interface language reverts from EN to HE automatically when navigating pages (Medium Priority).
